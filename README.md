# Nuvio Brief

Ежедневный утренний крипто-бриф в Telegram для новичков. Один пост в 10:00 MSK.
Логика по блокам ТЗ раскидана по `src/blocks/*`, оркестрация — `src/index.ts`.

## Локальный запуск

```bash
npm install
cp .env.example .env   # заполнить токены
npm start               # реальная отправка в Telegram
```

Проверка сборки сообщения без реальной отправки:

```bash
DRY_RUN=1 npm start
```

## Секреты

Добавить в GitHub → Settings → Secrets and variables → Actions:

- `TELEGRAM_BOT_TOKEN` — токен бота от @BotFather
- `TELEGRAM_CHANNEL_ID` — ID канала (бот должен быть в нём админом)
- `GROQ_API_KEY` — ключ console.groq.com (опционально — без него Блок 3 всегда
  использует нейтральный фолбэк, бриф всё равно уходит)
- `REF_LINK` — реф-ссылка для футера (опционально, по умолчанию `[ссылка]`)

## Расписание

`.github/workflows/daily-brief.yml` — cron `0 7 * * *` (= 10:00 MSK, UTC+3, без перевода
часов). Можно запустить вручную через "Run workflow" (workflow_dispatch).

## Идемпотентность

`state/last_sent_date.txt` хранит дату (MSK) последней успешной отправки. После
успешного запуска workflow коммитит эту дату обратно в репозиторий — повторное
срабатывание крона в тот же день не отправит бриф дважды.

## Банк советов (Блок 4)

`src/blocks/advice.ts` — 60 советов (21 [скам] / 21 [термин] / 18 [ошибка]),
ротация одна запись в день по дате MSK (детерминированно, без повторов подряд
в пределах длины банка). Категория — метаданные бота, в сам пост не попадает.

## Что не реализовано (см. §12 ТЗ)

- Новости в отборе события (Блок 3 работает только на движении цены BTC/ETH)
- Мост Этапа 1, бот розыгрыша Bybit — вне рамок MVP
